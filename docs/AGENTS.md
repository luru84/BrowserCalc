# AGENTS.md（LLM/自動エージェント向けガイド）

## 目的
- BrowserCalc のリポジトリでエージェント/LLMにタスクを依頼する際の共通ルールを示す。
- 依頼のブレを減らし、安全かつ再現性のある成果を得ることが狙い。

## 依頼時に必ず伝えること
- ゴールと受け入れ条件（テスト観点含む）。
- 関連ドキュメント: `docs/REQUIREMENTS.md`, `docs/DESIGN.md`, `docs/ARCHITECTURE.md`, `docs/PLAN.md`, `docs/CODING_GUIDELINES.md`, `docs/ADR.md` など該当箇所。
- 変更してほしくないファイルや制約（例: 既存の未コミット変更を壊さない、依存追加禁止など）。
- 実行してほしいコマンド（例: `npm run lint`, `npm run typecheck`, `npm test`）。実行不可な場合はその理由の報告を求める。
- 出力フォーマットの期待（変更点の要約、テスト結果、次のアクションなど）。

## 作業ルール
- 破壊的操作禁止: `git reset --hard`, 大量削除、無断のフォーマット全体適用は禁止。既存未コミット変更は保持する。
- 秘密情報や認証情報を入力しない。外部ネットワークや未知のAPIを呼ばない。
- 依存追加は事前合意と理由が必要。ランタイム依存を増やす場合はADR検討を行う。
- コードは `docs/CODING_GUIDELINES.md` に従い、UI/ロジック分離とアクセシビリティを維持する。
- テストは可能な範囲で実行し、実行できない場合は理由と代替確認を記載する。
- 変更内容はファイルパスを明示して要約し、テスト結果と既知の未対応事項を併記する。

## レスポンス期待値
- 何を変えたか（ファイル単位）と理由を簡潔に述べる。
- 実行したコマンドと結果、未実行の場合の理由。
- リスク・残課題があれば明記する。必要に応じて次のステップ提案を添える。
